# Two Sum

## Date
15/02/2026

## Approach
Used a brute-force nested loop.
For each index i, checked elements after it (j = i + 1)
to avoid duplicate pairs and using the same element twice.

## Time Complexity
O(n^2)

## Space Complexity
O(1)

## Key Learnings
- Controlled loop boundaries
- Avoided modifying loop counters
- Understood unique pair traversal
